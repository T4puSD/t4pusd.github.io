Thinks I Like About Hibernate

Hibernate is a extraordinary thing for Java developers. Mapping database rows into POJO seems pretty simple job but managing it’s lifecycle, capturing changes to an object is really hard job and hibernate makes those things really easy for the developers.

On top of hiberante Spring JPA helps us maintain RDBMS’s ACID property in check by maintaining database transaction with simple annotation. This kinds of things facinates me.

Even though hibernate makes our life easier it’s like an double edge sword. If we are not careful it can make our life hell. Walking on this thin line of do and don’t is a good sensation that drives me everyday.

Communication Technologies I Use

HTTP is a common communication techonlogy that we use everyday it is a protocol which greatly benefit the client and server communication. For example, browsing the internet with a browser where browser is the client and the server is the service hosted in some cloud provider or local system. It is a facinating inventions of humans which was standardized gradually.

Microservices is a huge buzz word in recent tech industrise which created the need of massive server to server communication between multiple stateless server deployed in clusters to ensure zero downtime and high availability. Using HTTP during a server to server communication required statefull syncronous communication between two servers which was hindering the usefullness of microservices and limiting their capability of being decoupled from each other. I am amazed with the messaging broker communication technique which is getting used to operate complex microservice architechtures.

RabbitMQ, Kafka are some technogoly stack I used which leverages the broker functionality to perform communication between multiple services asynchronously.
